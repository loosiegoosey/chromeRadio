chromeRadio Version 1.0

chromeRadio is an Extension for the Google Chrome Browser developed for the GTUG Battle event in Munich, Germany.

How To Use chromeRadio

chromeRadio is an extension that allows you to discover audio-files on websites you visit in seconds and play them instantly back using the Chrome Browser's built in HTML5 capabilities. It also allows you to store them in a music library and organize them using categories.

Let us show you how it works. For this brief tutorial we assume the extension has been correctly activated and you see the icon in your browser-bar.

Step 1: Go to a page that has direct links to either .mp3 or .ogg files. youlooknicetoday.com is the website of a (great) podcast and we will use it in this example. As soon as the page finishes to load you should see a tiny badge appear on top of the extension-icon notifiying you of the number of files that have been found by looking at the page's HTML. It should look like this:

Please note that this might not work 100% of the time, because of certain limitiations withing the extension API. If it doesn't it often helps to simply click on another tab and then click again on the tab that contains the music files.

Step 2: When you click the icon you will be presented a popup that lists all the music files that have been found. In our example it's ten files. You will initially see the anchor text of the links, but since that is often not very helpful, a click on "Toggle View" will show you the actual filenames. You can now either playback those files using the "Play Instantly"-Link, or you can add them to your library so you can listen to them later. This is how it should look like on youlooknicetoday.com:

Step 3: Ok, so in the next step we open the library. For this tutorial I have a library with 10 tracks from youlooknicetoday.com and 3 tracks from PMA.

You can export and import files through the JSON-Format. The functionality is included in the drop-down box. If you delete a category, all contained files will be classified as "Uncategorized".

Finally here are some of our ideas for future development:

  • Import of RSS-feeds for podcasts with ability to automatically add to the library and notify the user.
  • Sorting capabilities.
  • Support for other formats, such as videos or streaming radio.
  • A server application that allows for simple discovery and recommendations through the extension.
  • Drag and drop functionality for the gui.
  • Multiple themes that users can choose from and ability to create new ones.
